# Stash — Design Context

## Users
People who live in their Mac's menu bar all day — developers, makers, knowledge workers — who want a single warm corner of the OS for the small jobs that break flow: grabbing something they copied, jotting a note, capturing a task, snapping a window, expanding a snippet. Used in quick bursts dozens of times a day, often without leaving the keyboard.

## Brand Personality
**Warm · tactile · unhurried-but-fast.** Stash is the opposite of a cold dev tool. It should feel like a well-made physical desk — paper notes, a terracotta pen cup, frosted glass — not a dashboard. Three words: **warm, considered, quick.** Emotional goal: calm confidence. The user should feel the app *cares about the small stuff* so they don't have to.

## Aesthetic Direction
- **Theme: native-adaptive (warm).** Follows the macOS system appearance — **warm cream/sand in Light**, **warm-neutral dark (`#201E1B` panel) in Dark**, terracotta accent in both. _(Decision 2026-06-19: the user chose native-adaptive over the original light-only intent, so the app feels like a real macOS app on a dark Mac. The menu-bar popover AND the Notes/Tasks windows both adapt; the brand lives in the terracotta accent + paper chips + SF Rounded, not in forcing a single theme.)_ Never clinical white; never "dark mode with glowing neon accents." Dark surfaces are warm-tinted, not cold gray.
- **Concept: "a warm physical desk."** Paper (sticky notes), frosted glass (the hub), terracotta as the one bold accent. Tactile detail over flat minimalism.
- **References (feel, not copy):** Things 3's calm hierarchy, Bear's warmth, **Raycast's speed + information density** (esp. for clipboard history — rich type-aware rows, metadata, instant feel).
- **Anti-references:** generic admin dashboards; left-border accent stripes; glassmorphism-everywhere; neon-on-dark; cramped flat lists; anything that reads "AI made this."
- **Native first:** this is AppKit/SwiftUI on macOS — it MUST feel native. Use the **SF family**, not web fonts. Windows use **`NavigationSplitView` + `.listStyle(.sidebar)`** (real vibrancy sidebar, unified titlebar, native toolbar) — NOT hand-built `HSplitView` with flat fills. Colors come from appearance-aware **tokens** (`Color(light:dark:)`) or SwiftUI semantic colors (`.primary`/`.secondary`), never hardcoded cream/near-black that breaks in the other mode.

## Type (native, not web)
- **Display / titles:** SF Pro **Rounded** — friendly, warm, slightly soft. Note titles, card headers, the app's voice.
- **Body / UI:** SF Pro **Text** — refined, legible at small sizes.
- **Mono:** SF **Mono** — only for snippet triggers (`:sig`), code clipboard items, and hex codes. Never as lazy "techy" shorthand.
- **Hierarchy (fixed rem-like pt scale, ≥1.25 contrast):** title 20–22 / item-title 14 / body 13 / meta 11 / section-label 10 all-caps tracked. Few sizes, strong contrast.

## Color (tint neutrals toward terracotta) — all neutrals are `Color(light:dark:)`
- **Accent (the rare 10%):** terracotta `#c8642f` — FIXED in both modes. Selection, active tab, pins, primary actions — sparingly.
- **Text:** primary `#2c2925` / dark `#F3EFE9`; secondary `#6b655c` / dark `#B5ADA3`; tertiary `#9a948a` / dark `#8D867C`. Never pure black/white.
- **Surfaces:** panel cream `rgba(252,250,246,.93)` / dark `#201E1B`@.96; surface tile light / dark `#2A2724`. Hairline dark@.06 / white@.09. Row-selected = accent@.10 / @.16. Warm neutrals tinted toward terracotta; never clinical.
- **Kind colors (clipboard):** link = muted blue, image = violet-gray, code = slate, color-swatch = the literal color, text = warm gray.
- **Sticky pastels:** yellow `#fdf0c2`, peach `#fcdcc6`, blue `#d4e4f2`, mint `#d9ecda` (accents `#c8642f / #b97a4a / #5b86b8 / #5e8a52`).
- **Status dots:** running `#3fa45b`, waiting `#d8a13a`, idle `#a39a8c`. **Priority:** high `#c8642f`, med `#d8a13a`, low `#b3a99b`.

## Design Principles (guide every decision)
1. **Warmth is the differentiator.** Cream + terracotta + paper. If a surface feels clinical, it's wrong.
2. **Tactile, not flat.** Paper color chips, custom checkboxes, soft selection tints, kind-aware previews — small physical touches that say "made by hand."
3. **Dense but breathing (Raycast lesson).** Lists are information-rich (type, title, preview, metadata, actions) yet have rhythm — generous section spacing, tight intra-row spacing.
4. **Hierarchy through type weight + space, not boxes.** Few sizes, clear contrast, varied spacing. No card-in-card, no border stripes, no everything-centered.
5. **Fast feedback.** Hover reveals actions; selection is instant; copy/expand animate with quick ease-out (no bounce). Optimistic, never laggy.

## Banned (AI tells — never)
- Left/right colored border stripes on rows/cards (>1px). Use tints, leading tiles, or nothing.
- Gradient text.
- Glassmorphism used decoratively everywhere (the hub's frost is purposeful; don't add more).
- Generic rounded-rect + drop-shadow cards with no point of view.
- Monospace as "techy" decoration.
